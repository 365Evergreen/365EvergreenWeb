# 🔷 1. CTA → Drawer trigger (entry point)

### User story

**As a visitor ready to engage**,  
I want the CTA to open a guided interaction without leaving the page,  
so that the experience feels seamless and low effort.

### Acceptance criteria

* Clicking “Start a conversation” opens a drawer (not a new page)
* Drawer slides in from right (desktop) or bottom (mobile)
* Transition is smooth and quick (no abrupt jumps)
* Background page becomes visually subdued (overlay or dim effect)
* Focus shifts into the drawer (keyboard accessibility)

***

# 🔷 2. Drawer presentation (first impression)

### User story

**As a user opening the drawer**,  
I want to immediately understand what I’m about to do,  
so that I feel comfortable proceeding.

### Acceptance criteria

* Clear heading:
  * e.g. “Start a conversation”
* Short supporting text:
  * e.g. “Tell us a bit about your organisation”
* No overwhelming inputs on first view
* Close option is clearly visible (X or back control)
* Drawer width feels balanced (not full-screen on desktop)

***

# 🔷 3. Multi-step flow (guided interaction)

### User story

**As a user completing the form**,  
I want the process to be broken into simple steps,  
so that it feels easy and manageable.

### Acceptance criteria

* Form is split into logical steps (e.g. 3–5 steps)
* Each step contains a small number of inputs
* Only one step is visible at a time
* Progression feels linear and clear
* No cognitive overload on any step

***

# 🔷 4. Progress indicator story

### User story

**As a user moving through the form**,  
I want to know how far I’ve progressed,  
so that I feel in control and motivated to continue.

### Acceptance criteria

* Progress indicator visible (e.g. step counter or progress bar)
* Clearly shows:
  * current step
  * total steps
* Updates on each step transition
* Does not dominate the interface

***

# 🔷 5. Navigation between steps

### User story

**As a user completing the form**,  
I want to move forward and backward easily,  
so that I can correct or review responses.

### Acceptance criteria

* “Next” button advances the step
* “Back” button allows previous step navigation
* Inputs are preserved when navigating back
* No data loss during navigation
* Buttons are consistently positioned

***

# 🔷 6. Input interaction story

### User story

**As a user entering information**,  
I want inputs to feel clear and easy to use,  
so that I can complete the form without friction.

### Acceptance criteria

* Inputs have clear labels (not placeholders only)
* Focus state is visible and accessible
* Inputs are appropriately sized (especially mobile)
* Field types match intent (text, select, etc.)
* Validation is gentle and non-intrusive

***

# 🔷 7. Tone and experience story

### User story

**As a potential client**,  
I want the form to feel like a conversation,  
so that I am not discouraged by a traditional form experience.

### Acceptance criteria

* Language is conversational and human
* No technical or bureaucratic wording
* Prompts feel like guidance, not interrogation
* The experience aligns with the site tone

***

# 🔷 8. Step completion flow

### User story

**As a user progressing through steps**,  
I want a sense of momentum,  
so that I am encouraged to complete the process.

### Acceptance criteria

* Each step transition is smooth (slight animation)
* Completion of a step feels acknowledged (e.g. subtle transition)
* No unnecessary delays between steps
* The overall flow feels quick and efficient

***

# 🔷 9. Submission story

### User story

**As a user finishing the form**,  
I want to submit my details easily,  
so that I can initiate contact without frustration.

### Acceptance criteria

* Final step clearly indicates submission
* Primary button label reflects intent:
  * e.g. “Send message” or “Start conversation”
* Submission is immediate (no confusing delays)
* Loading state is visible if needed

***

# 🔷 10. Success state story

### User story

**As a user who has submitted the form**,  
I want confirmation that my request was received,  
so that I feel confident the process worked.

### Acceptance criteria

* Clear success message displayed
* Message is reassuring and professional:
  * e.g. “Thanks — we’ll be in touch”
* Next step is communicated (e.g. response expectation)
* Option to close the drawer is available
* Form is reset or hidden after submission

***

# 🔷 11. Exit / abandonment story

### User story

**As a user unsure about proceeding**,  
I want the ability to exit easily,  
so that I don’t feel trapped in the interaction.

### Acceptance criteria

* Drawer can be closed at any point
* Close action is clearly visible
* No forced completion of form
* Optional:
  * gentle confirmation if partly completed

***

# 🔷 12. Accessibility story

### User story

**As a user with accessibility needs**,  
I want the drawer and form to be fully usable,  
so that I can complete it without barriers.

### Acceptance criteria

* Focus is trapped within the drawer while open
* Escape key closes the drawer
* All controls are keyboard navigable
* Proper labels for assistive technologies
* Contrast meets accessibility standards

***

# 🔷 13. Mobile responsiveness story

### User story

**As a mobile user**,  
I want the drawer experience to adapt cleanly,  
so that I can complete the form comfortably.

### Acceptance criteria

* Drawer appears as bottom sheet or full-height panel
* Inputs are large and touch-friendly
* Buttons are full-width or easily tappable
* Layout avoids crowding or overflow
* Keyboard interactions don’t break layout

***

# 🔷 14. Performance story

### User story

**As a visitor opening the form**,  
I want the interaction to feel fast and responsive,  
so that I don’t lose confidence.

### Acceptance criteria

* Drawer opens instantly or with minimal delay
* No lag when switching steps
* Lightweight animations (no jank)
* Form state is managed efficiently

***

# 🔷 15. Brand experience story (critical)

### User story

**As a potential client**,  
I want the interaction to feel consistent with the brand,  
so that I trust the quality of the service.

### Acceptance criteria

* Colours follow the Evergreen design system
* Motion is subtle and calm (no flashy effects)
* Layout is clean and uncluttered
* Typography is consistent with the rest of the site

***

# 🔷 16. Data handling reassurance story

### User story

**As a user sharing information**,  
I want reassurance that my data is handled respectfully,  
so that I feel comfortable submitting.

### Acceptance criteria

* Optional short note:
  * “We’ll only use this to respond to your enquiry”
* No aggressive data capture language
* No unnecessary fields
* Tone remains respectful and transparent

***

# ✅ What you now have

This gives you a **complete interaction system**, not just a form:

* CTA → triggers
* Drawer → contains experience
* Multi-step → reduces friction
* Submission → closes loop
* Tone → stays aligned with brand
