# 🧱 Latest Posts Component – User Stories

***

## 🔷 1. Core component story

### User story

**As a potential client**,  
I want to see recent insights or content,  
so that I can assess expertise and credibility before engaging.

### Acceptance criteria

* Section is clearly labelled (e.g. “Latest insights” or “Latest posts”)
* Displays a small number of recent items (3–4 max)
* Content appears current and relevant (not stale)
* Section supports scanning rather than deep reading
* Positioned before or near final CTA

***

## 🔷 2. Content relevance story

### User story

**As a business decision-maker**,  
I want to quickly understand the value of the content,  
so that I can judge whether this organisation understands my needs.

### Acceptance criteria

* Each post has:
  * title
  * optional short description/excerpt
* Titles are outcome/problem-focused (not technical)
* Topics align with:
  * clarity
  * information structure
  * M365 outcomes (not tools-first)
* No overly technical titles

***

## 🔷 3. Card layout story

### User story

**As a scanning user**,  
I want content displayed in a clean and consistent format,  
so that I can browse easily.

### Acceptance criteria

* Cards follow consistent layout
* Each card includes:
  * title
  * optional metadata (date or category)
  * subtle hover state
* Cards are evenly spaced
* Avoid visual clutter (no heavy thumbnails unless intentional)

***

## 🔷 4. Interaction story

### User story

**As a user interested in learning more**,  
I want to easily navigate to full articles,  
so that I can explore topics in more depth.

### Acceptance criteria

* Cards are clickable (entire surface or clear link)
* Link behaviour is predictable (no unexpected redirects)
* Hover interaction provides subtle feedback
* Loading or navigation is smooth

***

## 🔷 5. Trust-building story

### User story

**As a cautious buyer**,  
I want to see evidence of thought leadership,  
so that I feel confident in the expertise.

### Acceptance criteria

* Content tone reflects:
  * clarity
  * insight
  * practical thinking
* Posts reinforce positioning (not generic blog content)
* Section supports credibility without overwhelming user

***

## 🔷 6. Content lifecycle story

### User story

**As a returning visitor**,  
I want to see updated content over time,  
so that I feel the business is active and engaged.

### Acceptance criteria

* Section is connected to dynamic content source (CMS or markdown)
* New posts automatically surface in this section
* Layout supports varying content length (no truncation issues)

***

## 🔷 7. Optional “view more” story

### User story

**As a content-oriented visitor**,  
I want access to all articles,  
so that I can explore further.

### Acceptance criteria

* Optional link:
  * “View all insights” or similar
* Link is visually secondary to cards
* Leads to blog/index page

***
