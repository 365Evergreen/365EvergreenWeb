# 🧱 Problem Component – User Stories

***

# 🔷 1. Core Section Story (overall experience)

### User story

**As a potential client**,  
I want to quickly recognise that my current workplace setup is inefficient,  
so that I feel understood and motivated to explore a solution.

### Acceptance criteria

* The heading “Work shouldn’t feel this hard” is immediately visible without scrolling
* The section communicates pain points without technical jargon
* The content resonates with common M365 frustrations
* The section creates a subtle sense of discomfort (recognition, not fear)
* The user can scan the section in under 5 seconds

***

# 🔷 2. Card 1 – “Files everywhere”

### User story

**As a user searching for information at work**,  
I want to recognise the frustration of scattered files,  
so that I feel this service understands my daily challenges.

### Acceptance criteria

* The title clearly communicates the issue in 1–2 words
* Supporting text reinforces the problem without over-explaining
* The user can immediately associate with “can’t find what you need”
* The card uses minimal UI elements (no clutter, no icons overload)
* The message is relatable across industries (not niche)

***

# 🔷 3. Card 2 – “Manual processes”

### User story

**As someone performing repetitive tasks**,  
I want to see my inefficiencies reflected back to me,  
so that I recognise the value of automation.

### Acceptance criteria

* The phrase “Manual processes” is clearly visible and prominent
* Supporting text (“Constant repetition”) reinforces time inefficiency
* The wording avoids technical solution language (e.g. no “Power Automate”)
* The card contributes to a growing sense of friction (but remains calm in tone)
* The user understands this is a time and productivity issue

***

# 🔷 4. Card 3 – “Disconnected tools”

### User story

**As a user dealing with multiple systems**,  
I want to recognise the frustration of tools not working together,  
so that I see the need for integration.

### Acceptance criteria

* The title “Disconnected tools” is immediately understandable
* Supporting text (“Nothing syncs”) is concise and impactful
* The user can relate this to common M365 overlaps (Teams, SharePoint, email)
* The card avoids technical explanation of integration
* The problem feels familiar, not abstract

***

# 🔷 5. Section Conclusion Statement

### User story

**As a potential client**,  
I want a simple explanation for why these problems exist,  
so that I feel reassured this is normal and solvable.

### Acceptance criteria

* The statement “Most organisations outgrow their setup” is clearly visible
* The tone feels reassuring, not critical or alarmist
* The user feels:
  * “This isn’t just me”
  * “This happens naturally”
* The message transitions smoothly toward the solution section
* No blame language is used

***

# 🔷 6. Emotional Journey Story (critical for conversion)

### User story

**As a visitor landing on the site**,  
I want to feel progressively understood as I scan the problems,  
so that I continue scrolling instead of leaving.

### Acceptance criteria

* The cards are ordered logically:
  1. Files (easy entry point)
  2. Processes (deeper friction)
  3. Systems (structural issue)
* Each card increases recognition without overwhelming the user
* The section does not introduce solutions yet
* The tone remains calm, not urgent or aggressive

***

# 🔷 7. Accessibility & clarity story

### User story

**As a user with varying levels of technical knowledge**,  
I want this section to be easy to read and understand,  
so that I can engage with the content without effort.

### Acceptance criteria

* Text contrast meets accessibility standards
* Cards are readable at a glance (no long paragraphs)
* Language avoids technical jargon completely
* Layout works clearly on mobile (cards stack vertically)
* Headings are scannable (clear hierarchy)

***

# 🔷 8. UI / interaction story

### User story

**As a user interacting with the page**,  
I want subtle visual feedback as I explore the problem section,  
so that the experience feels modern and engaging.

### Acceptance criteria

* Cards have a subtle hover effect (lift + shadow)
* Transitions are smooth and not distracting
* No heavy animation that distracts from reading
* Visual hierarchy is consistent across all three cards

***

# 🔷 9. Content tone story (important for your brand)

### User story

**As a business decision-maker**,  
I want the messaging to feel professional and measured,  
so that I trust the expertise behind it.

### Acceptance criteria

* No exaggerated claims or dramatic language
* Tone is calm, confident, and observational
* Language feels like insight, not marketing
* Content aligns with a consulting-style voice
