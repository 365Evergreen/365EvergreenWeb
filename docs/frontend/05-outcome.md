# 🌿 Outcome Component – User Stories

***

## 🔷 1. Core Section Story (overall outcome framing)

### User story

**As a potential client**,  
I want to understand what a better workplace looks like,  
so that I can visualise the benefits of improving my current setup.

### Acceptance criteria

* The headline “Imagine a workplace that just flows” is immediately clear
* The section shifts tone from problem → positive without feeling unrealistic
* The benefits are described in everyday language (not technical)
* The user can quickly grasp the outcome in under 5–7 seconds
* The section feels calm and aspirational, not exaggerated

***

## 🔷 2. Outcome Point – “Everything has a place”

### User story

**As a user searching for information**,  
I want to feel that content can be structured logically,  
so that I believe finding information can be simple.

### Acceptance criteria

* The message reinforces clarity and organisation
* The wording is concise and universally relatable
* No mention of tools (e.g. SharePoint)
* The benefit is intuitive without explanation

***

## 🔷 3. Outcome Point – “Processes run automatically”

### User story

**As someone currently doing repetitive work**,  
I want to see a future where manual effort is reduced,  
so that I understand the value of automation.

### Acceptance criteria

* The phrase clearly communicates reduced effort
* The benefit focuses on time and ease, not technology
* No mention of “automation tools” or specific features
* The benefit feels realistic (not “fully automated everything”)

***

## 🔷 4. Outcome Point – “Systems support your team”

### User story

**As a user interacting with multiple systems**,  
I want to feel that tools can work together effectively,  
so that my work feels smoother and more efficient.

### Acceptance criteria

* The message implies integration without explaining it
* The language remains human-focused (team, work, flow)
* The benefit resonates across different organisations
* Avoid technical terminology (e.g. APIs, connectors)

***

## 🔷 5. Visual / motion story (Evergreen lead-in)

### User story

**As a visitor**,  
I want subtle visual reinforcement of growth and flow,  
so that the outcome feels natural and connected to the brand.

### Acceptance criteria

* A thin green line animates gently across the section
* Animation does not distract from reading
* Motion reinforces progression (left → right or subtle expansion)
* The animation feels calm and purposeful

***

## 🔷 6. Emotional journey story

### User story

**As a visitor progressing through the page**,  
I want to feel a shift from frustration to possibility,  
so that I continue toward the solution.

### Acceptance criteria

* The section contrasts clearly with the problem section
* Tone is optimistic but grounded
* No pressure or urgency language is introduced
* The user feels a sense of relief rather than excitement alone

***
